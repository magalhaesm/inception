Project Description

This project is a small infrastructure composed of different services, each running in its dedicated Docker container. The containers are built from the penultimate stable version of Alpine or Debian for performance reasons. The services include NGINX with TLSv1.2 or TLSv1.3, WordPress with php-fpm, and MariaDB. Volumes are used to store the WordPress database and website files, and a docker-network is established to connect the containers. All containers automatically restart in case of a crash.

Infrastructure Structure

The project consists of the following components:

  • NGINX Container with TLSv1.2 or TLSv1.3
  • WordPress + php-fpm Container
  • MariaDB Container
  • Volume for WordPress Database
  • Volume for WordPress Website Files
  • Docker Network for Inter-Container Communication

Bonus Features

In addition to the main project, the following bonus features were included:

  • Set up Redis cache for your WordPress website
  • Set up an FTP server container pointing to the volume of your WordPress website
  • Create a simple static website in the language of your choice (excluding PHP)
  • Set up Adminer
  • Set up a service of your choice (justify your choice during the defense)
